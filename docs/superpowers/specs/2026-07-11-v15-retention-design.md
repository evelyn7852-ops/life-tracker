# V1.5 留存四阶段：积累仪表盘 + 旅行闭环 + 活的学习流 + 狗随记录

> 用户批准 2026-07-11，四阶段全选，按序执行、逐阶段上线。核心目标：补习惯循环的「奖励/积累」半环 + 闭合「计划↔现实」。

## 阶段①：积累仪表盘 + streak（最先，数据现成）
- **streak**：连续记录天数（截至今天、每天 ≥1 条 entry 即算）。断链归零。首页显眼位（心情行下方或日历上方）「🔥 连续 N 天」。
- **本月各域统计**：饮食/运动/旅行/阅读/日记/学习 本月条数，小色块横排（复用域点色）。
- **情绪趋势**：近 14 天 mood emoji 一行 sparkline（journal 有 mood 的按日取最新，缺失留空点）。
- **年度积累计数**：今年总记录数、本月阅读条数（reading）、本月训练次数（workout done）。旅行「今年国家数」——真实 travel 记录无结构化国家字段，本期改用 travelPlan 中 status=done 的行程计数，标注来源。
- 位置：首页新增紧凑「统计」卡（日历下方、规划上方）；周览页顶部加 streak + 本月计数条。
- 纯读现有 entries/workouts，无新表。数据量优化：首页统计只拉本月+近14天范围，不全量。
- TDD：streak 计算（含断链/今天未记/跨月）、本月分域计数、mood sparkline 取值。

## 阶段②：旅行可编辑 + 计划↔现实闭环
- travelPlan.json → Supabase 表 `trips`（迁移 004，字段照 travelPlan.json + user_id + RLS）。首次登录做一次性种子导入（把 152 行程写入当前用户，幂等：已有则跳过）。
- 旅行视图从只读→可编辑：改目的地/时段/预算/天数/状态、增删行程、拖动改年（简单 select 改年即可，不做拖拽）。
- 闭环：travel 域 entry 记录时，可关联到某条 trip（可选）；trip 到期（当年当季）在首页提醒「本季有旅行计划：X」；trip 可从「已订/已完成」手动标记，或由关联的真实记录自动标 done。
- 「往前提自动调整」：改某 trip 年份后，同年冲突用黄标提示（不自动重排，给你决策）。
- TDD：种子幂等、trip CRUD、到期提醒逻辑。

## 阶段③：活的学习流 + 推送
- 黑客松固定页保留为「集训存档」，另建「学习流」：新表 `learning_items`（推荐的课程/文章/资源，字段 title/url/source/tag/status/added_at/added_by）。
- 定时 cron（Supabase scheduled function 或 GitHub Actions cron）每周调 LLM + 检索，追加 2-3 条新 AI 学习资源到 learning_items，去重。
- Web Push 基建：service worker push 订阅 + 后端存订阅 + cron 触发推送「本周新增 3 个值得学的 AI 资源」。用户可开关。
- 学习流视图：待读/在读/已读三态，勾选进度入库（替代 localStorage 单设备限制）。
- 大阶段，可再拆子期（先视图+手动加，后 cron+push）。

## 阶段④：狗随记录变 + 小乐趣
- DogBanner 从纯随机 → 反映当日记录：记饮食→吃🦴、记训练→跑、记阅读→趴着看书、一天没记→睡💤、心情😢→安慰状。
- 读当日 entries 域集合决定狗状态权重（仍保留随机切换，只是偏向今日活动）。
- 其他微乐趣：streak 里程碑（7/30/100 天）小庆祝动画；空态文案更有梗。
- 最轻，纯前端。

## 通用验收
每阶段：全量单测绿 + tsc + build + 375px 实测 + 逐阶段独立上线。②③需用户跑 SQL 建表（届时给）。③的 push 需用户授权通知权限。
