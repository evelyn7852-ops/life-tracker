# V1.3：首页日历 + 节气节日 + 狗 banner + 地点化每日一句 + 周览周日起始

> 用户需求 2026-07-09 晚批。

## A. 周览修正（先行，影响数据边界）
- 周起始改为**周日**（现为周一）：weekStart() 及 summarize 边界随之变；缓存 key（period_start）自然失效重算，无迁移
- 周览标题标注范围：「周览 7月5日–7月11日」

## B. 首页日历
- 首页日期行升级：**2026年7月9日 周四**（含年份）
- 新增月历网格组件 CalendarView（首页内嵌，当月视图，可左右切月）：
  - 有记录的日期加小圆点（按域颜色，最多 3 点 + 溢出记号）
  - **节假日标记**：阳历节日（元旦/情人节/妇女节/劳动节/儿童节/国庆/平安夜/圣诞等）+ 农历节日（春节/元宵/端午/七夕/中秋/重阳，2026-2030 内置查表，出处核对）+ **24 节气**（寿星公式计算 + 2026 锚点单测：立春/清明/夏至/冬至等）
  - 节气/节日显示对应 emoji（🌱立春 🌸清明 ☀️夏至 ❄️冬至 🧧春节 🎄圣诞…），与季节视觉对应
- **点日期 → 日详情**：弹层或子视图显示该日全部记录（复用 HistoryView 的按日查询），支持 ←/→ 前后翻日

## C. 每日卡升级
- 图片下方显示**地点**：解析 daily-image copyright（如「在沙巴的水稻田，老街，越南」→ 显示原文即可，不强行抽取）
- 每日一句去「全古诗词」化：daily-image Edge Function 扩展为 daily-card——同请求让 LLM 基于当日图片地点/季节生成一句短句（中文 ≤40 字，旅行/生活/自然向，不署名不引用以避瑞造），随图缓存（localStorage 按日 + 函数内不重复调用 LLM：结果存 summaries 表？不——新增轻量表不值，缓存放客户端 localStorage 按日即可，同日多设备各自调用可接受）
- 语录库保留为**离线/失败兜底**（90 条已核对）

## D. 狗 banner
- 首页底部固定矮条（高 ~56px）：像素/极简风 CSS 小狗
- 行为状态机随机切换（每 8-20s）：跑动（左右横穿）/ 打盹（趴下 zzz）/ 吃东西 / 挠痒；纯 CSS keyframes + emoji/SVG 组合，不引外部资源
- 尊重 prefers-reduced-motion：减动效时静态趴窝
